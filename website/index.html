Secure Tunneling Service

NXpose is a powerful Go-based service that allows you to expose local services to the internet through encrypted tunnels securely and easily.

Features

Why choose NXpose for your tunneling needs

Secure Encrypted Tunnels

All traffic is protected using TLS encryption to ensure your data remains private and secure.

Instant Exposure

Quickly expose any local service to the internet with a simple command.

Webhook Testing

Perfect for testing webhook integrations with your local development environment.

Multi-protocol Support

Works with HTTP, HTTPS, TCP and more protocols to support various applications.

Custom Subdomains

Generate random or specify your own subdomains for easy access to your services.

Reliable Connections

Automatic reconnection and efficient data transfer with minimal overhead.

Installation

Multiple ways to install NXpose

Pre-built binaries for Linux, macOS, and Windows are available on the releases page.

Simply download the appropriate binary for your platform, make it executable (if on Linux/macOS), and you're ready to go!

# Linux/macOS
chmod +x nxpose
./nxpose

# Windows
nxpose.exe

For Linux, APK and RPM packages are available for both AMD64 and ARM64 architectures:

# Install from RPM (Fedora, CentOS, RHEL)
sudo rpm -i nxpose_version_x86_64.rpm

# Install from APK (Alpine Linux)
sudo apk add --allow-untrusted nxpose_version_x86_64.apk

Build from source using either Make or the provided build script:

# Clone the repository
git clone https://github.com/nxrvl/nxpose.git
cd nxpose

# Using Make (Linux/macOS)
make build

# Using Windows build script
build.bat build

# The binaries will be available in the './bin' directory

If you have Go installed, you can use the Go install command:

go install github.com/nxrvl/nxpose/cmd/client@latest

Usage

How to use NXpose

Register with a Server

Before creating tunnels, you need to register with the NXpose server to obtain certificates:

nxpose register
Expose a Local Service

To expose a local service running on port 3000 via HTTP:

nxpose expose http 3000

This will create a secure tunnel and provide you with a public URL that forwards to your local service.

Skip Local Service Check

When testing webhooks or scenarios where you need to get the URL before starting your local service:

nxpose expose http 3000 --skip-local-check

This will create a tunnel without checking if the local service is running, which is useful when you need the public URL to configure your local application.
