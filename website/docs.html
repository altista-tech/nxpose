Architecture

NXpose consists of two main components:

  1. Client: Runs locally on your machine, connecting to your local service and the NXpose server.
  2. Server: Public-facing component that accepts secure connections from clients and routes external traffic through the established tunnels.

The architecture follows these principles:

  • Secure communication using TLS
  • Efficient data transfer with minimal overhead
  • Reliable connections with automatic reconnection
  • Clean separation of concerns between components
Connection Flow
  1. Client connects to the NXpose server over a secure WebSocket connection
  2. Server assigns a unique subdomain or URL for the tunnel
  3. External requests to the assigned URL are forwarded to the client
  4. Client forwards the request to the local service
  5. Responses follow the reverse path back to the original requester

This design allows for seamless tunneling without requiring public IP addresses or complex network configurations.

Client Commands

The NXpose client provides several commands to manage your tunnels:

Register

Before creating tunnels, you need to register with the NXpose server to obtain certificates:

nxpose register

Options:

  • --server - Specify a custom server address (default: nxpose.example.com)
  • --email - Email address for registration
Expose

Create a tunnel to expose a local service:

nxpose expose [protocol] [local-port] [options]

Examples:

# Expose HTTP service on port 3000
nxpose expose http 3000

# Expose HTTPS service on port 8443
nxpose expose https 8443

# Expose TCP service on port 22
nxpose expose tcp 22

Options:

  • --subdomain - Request a specific subdomain
  • --hostname - Request a specific hostname
  • --skip-local-check - Skip checking if local service is online
  • --server - Specify a custom server address
List

List all active tunnels:

nxpose list
Version

Display the client version information:

nxpose version

Use Cases

Webhook Development

Test webhook integrations with external services by exposing your local development server:

  1. Start your local webhook handler on port 3000
  2. Run nxpose expose http 3000
  3. Configure the external service to send webhooks to your NXpose URL
  4. Receive and debug webhooks in real-time
Sharing Development Preview

Share your local development with clients or team members:

  1. Run your application locally (e.g., on port 8080)
  2. Run nxpose expose http 8080 --subdomain myproject
  3. Share the provided URL with your clients or team
  4. They can access your local development in real-time
IoT Device Access

Access IoT devices behind firewalls or NAT routers:

  1. Install NXpose on your IoT device
  2. Run nxpose expose tcp 22 to expose SSH
  3. Access your device securely from anywhere
Remote Demos

Create demonstrations without deploying to production servers:

  1. Run your demo application locally
  2. Expose it with nxpose expose http 3000
  3. Present your demo to clients or stakeholders
  4. No need for complex deployments

Security

NXpose implements several security measures to protect your tunnels:

  • TLS Encryption: All traffic between clients, servers, and users is encrypted using TLS.
  • Client Authentication: Clients are authenticated using certificates or tokens to prevent unauthorized access.
  • Rate Limiting: Protection against brute force attacks and DoS attempts.
  • Tunnel Isolation: Each tunnel is isolated to prevent cross-tunnel access.
Security Notice

Remember that NXpose exposes your local services to the internet. Only expose services that are intended to be publicly accessible. Avoid exposing sensitive internal services without proper authentication.

Troubleshooting

If you're having trouble connecting to the NXpose server:

  1. Check your internet connection
  2. Verify the server address with nxpose ping
  3. Ensure your client is using the latest version
  4. Check if your network blocks WebSocket connections
  5. Try with verbose logging: nxpose expose http 3000 --verbose

If NXpose can't connect to your local service:

  1. Ensure your local service is running properly
  2. Check that the port number is correct
  3. Try accessing the service directly (e.g., curl localhost:3000)
  4. Make sure your service is bound to 0.0.0.0 or 127.0.0.1
  5. Use --skip-local-check if you're starting the service after NXpose

If you're experiencing certificate-related problems:

  1. Re-register with the server: nxpose register --force
  2. Check certificate expiration with nxpose inspect-cert
  3. Delete the certificate directory and register again
  4. Ensure your system time is accurate (important for certificate validation)

API Reference

NXpose provides a RESTful API for programmatic control of tunnels. The API is available on the server at /api/v1.

Endpoint Method Description
/api/v1/tunnels GET List all active tunnels
/api/v1/tunnels POST Create a new tunnel
/api/v1/tunnels/{id} GET Get tunnel details
/api/v1/tunnels/{id} DELETE Delete a tunnel
/api/v1/status GET Get server status

For detailed API documentation, please refer to the API Reference on GitHub.
